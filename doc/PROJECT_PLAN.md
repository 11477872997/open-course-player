# 项目计划

## 产品目标

做一个开源的本地文件目录桌面播放器。

用户选择一个下载好的目录后，可以在左侧浏览目录树，在右侧播放视频，继续上次进度，并快速切换上一节/下一节。

## 非目标

- 不做 DRM 破解或加密媒体规避。
- 早期不做云同步。
- 不做全盘扫描。
- 默认不上传文件名、路径、播放历史或媒体信息。
- 不内置内容下载、采集或爬虫。
- 不分发许可证不明确的专有解码器或二进制文件。

## 0.1 - 最小可用版

目标：先把 `.ts` 播放器做可用。

- 选择本地文件目录。
- 渲染左侧目录树。
- 过滤和排序可播放媒体文件。
- 通过本地 HTTP 媒体服务和 `mpegts.js` 播放 `.ts`。
- 通过内置播放器播放 `.mp4`。
- 基础控制：播放、暂停、进度、音量、静音、全屏、倍速。
- 记住最近打开目录。
- 记住每个文件的播放进度。
- 对不支持格式给出清楚提示。

## 0.2 - 主流格式兜底

目标：让常见本地视频格式都能打开。

- 打包或发现本机 mpv。
- 增加 mpv 适配器，兜底 `.mkv`、`.avi`、`.flv`、`.mov`、`.wmv`、`.rmvb`、`.vob` 和内置播放器播放失败的文件。
- 通过进程通信控制 mpv。
- 重点评估 EasyPlayer.js，用于 HLS、HTTP-FLV、fMP4、WebRTC、H.265 等增强播放场景。
- 自动加载同名字幕。
- 增加上一节/下一节。
- 增加最近播放和最近目录。

## 0.3 - 文件目录体验

目标：日常使用舒服。

- 当前文件夹播放列表。
- 目录树展开状态持久化。
- 继续观看入口。
- 快捷键。
- 播放错误诊断。
- 媒体信息面板。
- 可配置文件扩展名白名单。

## 0.4 - 打包和合规

目标：可以公开发布。

- Windows 安装包。
- GitHub Actions 构建流程。
- 第三方许可证清单生成。
- 发布产物校验和。
- 署名声明和源码提供规则检查。
- 可复用发布检查清单。

## 1.0 - 稳定开源版

目标：形成稳定可维护的公开项目。

- 完整架构和贡献文档。
- 稳定格式支持矩阵。
- 常见媒体文件手工测试矩阵。
- 尽量提供签名 Windows 安装包。
- 调研 macOS 和 Linux 多平台发布。

## 优先级规则

1. `.ts` 播放优先级最高。
2. 本地文件隐私优先于便利性。
3. 内置播放器负责快速路径，mpv 负责兼容路径。
4. 一个格式只有经过播放、拖动、暂停恢复和错误处理测试后，才能写成“已支持”。
5. 开源合规问题会阻塞发布。
