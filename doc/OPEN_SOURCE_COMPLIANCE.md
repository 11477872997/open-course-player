# 开源合规

本项目从一开始就按开源项目治理。播放器项目会涉及媒体引擎、解码器和打包二进制文件，许可证问题必须前置处理。

## 项目许可证

计划项目许可证：

```text
GPL-3.0-or-later
```

这个选择偏保守，原因是项目计划随桌面安装包分发 mpv/FFmpeg 相关播放组件。

如果未来版本不随包分发 mpv/FFmpeg，只使用宽松许可证的前端播放器库，可以重新评估许可证策略。在那之前，所有发布规划都按 GPL 义务处理。

## 依赖准入规则

新增依赖前必须记录：

- 包名或二进制名称。
- 版本。
- 来源地址。
- 许可证。
- 是链接、打包、运行时下载，还是可选依赖。
- 是否进入安装包。
- 是否有署名声明、源码提供或许可证副本要求。

许可证未知等于发布阻塞。

## 计划第三方组件

| 组件 | 预期许可证 | 计划用途 | 发布影响 |
| --- | --- | --- | --- |
| Tauri | MIT 或 Apache-2.0 | 桌面壳 | 保留许可证声明 |
| Vue | MIT | 前端框架 | 保留许可证声明 |
| Element Plus | MIT | UI 组件 | 保留许可证声明 |
| mpegts.js | Apache-2.0 | `.ts` 内置播放 | 保留许可证和署名声明 |
| hls.js | Apache-2.0 | HLS 播放 | 保留许可证和署名声明 |
| video.js | Apache-2.0 | 可选播放器界面 | 保留许可证和署名声明 |
| mpv | GPL-2.0-or-later | 原生播放兜底 | 分发时项目必须 GPL 兼容 |
| FFmpeg | 取决于构建，LGPL 或 GPL | mpv 媒体后端 | 必须确认构建参数和许可证模式 |
| EasyPlayer.js | 待确认 | 重点评估的增强播放器，覆盖 HLS、HTTP-FLV、fMP4、WebRTC、H.265 等场景 | 许可证确认前不能随包发布 |

## mpv 和 FFmpeg 注意事项

如果安装包内置 mpv：

- 包含 mpv 许可证文本。
- 记录 mpv 来源地址和版本。
- 提供对应源码获取方式或源码提供说明。
- 记录随 mpv 分发的 FFmpeg 构建许可证信息。
- 确保应用整体许可证 GPL 兼容。

不要从第三方二进制包里删除许可证文件。

## 署名声明规则

根目录 `NOTICE` 写项目层面的署名说明。

根目录 `THIRD_PARTY_NOTICES.md` 写依赖和二进制清单。每次发布前，只要依赖或安装包内容变化，就必须更新。

## 发布前合规检查

- 确认 `LICENSE` 存在。
- 确认 `NOTICE` 存在。
- 确认 `THIRD_PARTY_NOTICES.md` 是最新。
- 确认每个打包二进制都列入清单。
- 确认每个包的许可证都已知。
- 确认 EasyPlayer.js 没有在许可证未确认时进入公开安装包。
- 确认 GPL 覆盖的发布有对应源码可获取。
- 确认发布说明提到内置媒体引擎。

## 禁止事项

- 不分发没有明确授权的专有解码器。
- 不移除第三方许可证文件。
- 不声称支持 DRM 破解。
- 不隐藏安装包内置的原生媒体引擎。
- 不在许可证未知时发布二进制。
