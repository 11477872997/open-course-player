# 约束

这个文件定义项目能做什么、不能做什么。

## 平台约束

- Windows 是第一个支持平台。
- 桌面壳使用 Tauri 2。
- 前端基线使用 Vue 3 + TypeScript。
- Rust 负责本地文件访问、本地媒体服务和原生进程控制。
- macOS 和 Linux 必须经过打包和播放验证后，才能写成支持平台。

## 文件访问约束

- 应用只能索引用户明确选择的目录。
- 不做全盘扫描。
- 不静默读取无关用户目录。
- 后端读取或服务文件前必须规范化并校验路径。
- 本地媒体服务必须拒绝选中根目录之外的路径。
- 默认忽略隐藏文件和系统文件，除非用户在设置中开启。

## 播放约束

- `.ts` 是第一优先级格式。
- 大文件内置播放器播放必须通过支持分段请求的本地 HTTP 媒体地址。
- 不把 `file://` 作为默认播放路径。
- 内置播放器播放失败时必须能恢复，优先自动或手动切到 mpv。
- mpv 兜底必须作为独立播放引擎管理生命周期。
- 一个格式必须验证播放、拖动、暂停恢复和错误提示后，才能标记为完整支持。

## 本地媒体服务约束

- 只绑定 `127.0.0.1`。
- 默认使用随机可用端口。
- 支持 HTTP 分段请求。
- 尽量返回准确媒体类型。
- 只服务用户授权根目录下的文件。
- 界面错误信息尽量不暴露完整绝对路径。
- 应用退出时必须关闭服务。

## 隐私约束

- 默认不上传媒体文件、文件路径、目录名、播放历史或媒体元数据。
- 默认不启用遥测。
- 未来如果增加更新检查、崩溃上报或可选遥测，必须有文档说明并让用户可感知。
- 日志默认不要打印完整目录树或敏感本地路径，除非用户显式开启 debug 模式。

## 安全约束

- 本地媒体服务不能暴露到局域网网卡。
- 前端传入的绝对路径，后端不能直接信任。
- 不能把未清洗的用户输入拼进命令行外壳。
- 子进程优先使用结构化接口，不使用字符串拼接命令。
- mpv 进程通信必须限定在当前启动的进程上，并在播放结束后清理。
- 不实现 DRM 绕过或版权保护规避。

## UI 约束

- 第一屏就是播放器工作区，不做营销落地页。
- 核心布局是左侧目录树、右侧播放区。
- 长中文和英文文件名不能撑坏控件。
- 不支持的文件可以显示，但要有明确状态；除非用户开启过滤，不应静默消失。
- 错误提示要说明失败原因和下一步可尝试动作。

## 打包约束

- 每个打进安装包的二进制文件都必须记录来源、版本、许可证和获取/构建方式。
- 安装包内容必须和 `THIRD_PARTY_NOTICES.md` 对得上。
- mpv/FFmpeg 分发方式变化时，发布前必须重新做许可证检查。
- 发布产物应提供校验和。
- 不随包分发没有明确授权的专有解码器或素材。

## 开源约束

- 默认项目许可证为 `GPL-3.0-or-later`。
- 第三方许可证必须在发布前列清楚。
- GPL、LGPL、Apache、MIT 等许可证和署名声明必须保留。
- 如果加入 EasyPlayer.js，必须先确认许可证，再决定能否随包发布。
- 许可证未知的依赖视为发布阻塞。
